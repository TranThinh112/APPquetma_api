# Hướng dẫn đồng bộ dữ liệu TO lên Database API

## 1. Cấu trúc file PhanLoai (SAU REFACTOR)

### File Logic (xử lý dữ liệu):
  - create_to_logic.dart  → Logic tạo bao hàng (validate, quét mã, đóng TO)
  - scan_to_logic.dart    → Logic quét mã TO (validate, kiểm tra format)
  - table_to_logic.dart   → Logic danh sách TO (tìm kiếm, xóa, load data)

### File UI (giao diện):
  - create_to_screen.dart → Giao diện tạo bao hàng
  - scan_to_screen.dart   → Giao diện quét mã TO
  - table_to_screen.dart  → Giao diện danh sách TO

### File chung:
  - PhanLoaiScreen.dart   → Menu điều hướng 3 chức năng

### File giải thích:
  - giai_thich_logic_PhanLoai.txt → Giải thích chi tiết từng file logic


## 2. Tại sao cần đẩy dữ liệu TO lên API server?
- Đảm bảo dữ liệu bao hàng (TO) được lưu trữ tập trung.
- Cho phép các thiết bị khác truy cập, quản lý TO từ xa.
- Hỗ trợ chạy cả APP lẫn WEB (vì server online).

## 3. API endpoints cho bảng TO_orders

Server: https://server-production-fdce.up.railway.app

### POST /TO_orders
→ Upload 1 TO mới lên server
→ Body JSON:
{
  "maTO": "TO260325AB1X",
  "danhSachGoiHang": "SPXVN0612345678,SPXVN0687654321",
  "diaDiemGiaoHang": "HCM-Q1",
  "trangThai": "Packed",
  "packer": "dong",
  "ngayTao": "2026-03-25T13:00:00.000",
  "completeTime": "2026-03-25T13:30:00.000",
  "totalWeight": 8.5
}

### PUT /TO_orders/{maTO}
→ Cập nhật TO đã có trên server

### GET /TO_orders
→ Lấy tất cả TO từ server

## 4. Code trong api_service.dart

Các hàm đã thêm:
  - ApiService.uploadTO(TOModel to)        → POST 1 TO
  - ApiService.updateTOOnServer(TOModel to) → PUT cập nhật TO
  - ApiService.getAllTOsFromServer()         → GET tất cả TO

## 5. Lưu ý
- Bảng TO trên server tên là TO_orders (Thịnh đã tạo sẵn).
- Khi tạo TO xong (Complete) → app tự push lên server.
- Database SQLite local vẫn giữ để app offline hoạt động.
- Khi chạy trên WEB → không dùng SQLite, chỉ gọi API.
